Movie / TV Watchlist
CRM-Style Project

Build a record management system (mini CRM) where users manage movies and TV shows — create, edit, filter, validate, and store structured data in the browser.

HTML CSS Vanilla JS localStorage

You are not building a "movie app." You are building a record management system (mini CRM). Each movie = a structured data record.

4 JavaScript Tasks

✅ Start only after HTML + CSS is complete.

Phase 1 Form Engine form.js
  • Task 1: Show Rating & Notes ONLY when Status = Completed
  • Task 2: Validation — Title (required, min 2 chars), Genre (required), Poster URL (valid URL), Notes (max 500 chars)
  • Task 3: Show inline error messages under each invalid field
Phase 2 Star Rating Component rating.js
  • Hover → highlights stars up to hovered star
  • Click → saves the rating value
  • Hover out → keeps selected rating highlighted
  • Allow changing the rating by clicking again
Phase 3 Data Layer (Mini Database) storage.js
  • getAll() — return all records
  • save(record) — add new record
  • deleteRecord(id) — remove by id
  • update(id, data) — edit existing record ← REQUIRED
Phase 4 Rendering ui.js
  • Render all records dynamically on index.html
  • Show empty state message if no records exist
  • Update UI without reloading the page
  • ui.js handles all DOM updates — app.js never touches DOM directly
Phase 5 Full CRUD Operations app.js

Phase 6 Search, Filter, Sort app.js
  • Search: Real-time filtering as user types
  • Filters: By type (Movie / TV), by status
  • Sort: A–Z, Z–A, Highest Rated, Recently Added
Phase 7 CRM Features form.js
  • Lock Feature: If status = Completed → Title, Genre, Type become disabled (read-only)
  • Status Rule: Cannot mark as Completed without setting a rating first
